# Memories — Active Knowledge Base

This folder contains distilled, up-to-date facts about the SplitBrain.AI project.
Each file is a focused knowledge unit. Read these **before** any planning or implementation session.
These are living documents — update them when decisions change, not just when new things are added.

## Index

| File | Contents |
|---|---|
| `architecture-current-state.md` | Actual project structure, naming, what is and isn't implemented |
| `design-decisions.md` | Locked decisions with rationale (charting library, dark theme palette, etc.) |
| `review-cross-reference.md` | Deep-research review claims vs. actual codebase truth |

## How to Use

1. Read all files in this folder at the start of a new session.
2. After making significant architectural decisions, update the relevant memory file.
3. Do **not** add speculative or aspirational content here — that belongs in `Docs/Plans/`.
4. If a fact here contradicts `Plans/MasterPlanV4.md`, the Plans doc wins (it's the blueprint); update this file to reflect the gap.
